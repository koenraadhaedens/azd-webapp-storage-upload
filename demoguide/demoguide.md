## 🎯 Demo Overview

Audience: Technical (Azure architects, developers, security engineers)

Duration target: 30 minutes

Scenario objective: Demonstrate a production-style, OTP-gated upload workflow where:

- App Service authenticates users with short-lived OTP.
- Logic App sends OTP through Office 365 Outlook connector.
- Storage is private-only (no internet path), with RBAC data-plane access.
- Managed Identity replaces storage keys end-to-end.

## ⏱️ 5-Minute Pre-Demo Checklist

Run the following in PowerShell from scenario/secure-upload-portal.

~~~powershell
# 0) Confirm Azure context
az account show --query "{subscription:name, user:user.name}" -o table

# 1) Resolve resource group by naming convention
$rg = az group list --query "[?contains(name,'secureupload')].name | [0]" -o tsv
if ([string]::IsNullOrWhiteSpace($rg)) { throw "Resource group for secureupload not found." }
"Resource group: $rg"

# 2) Resolve key resource names
$web = az resource list -g $rg --resource-type Microsoft.Web/sites --query "[0].name" -o tsv
$logic = az resource list -g $rg --resource-type Microsoft.Logic/workflows --query "[0].name" -o tsv
$storage = az resource list -g $rg --resource-type Microsoft.Storage/storageAccounts --query "[0].name" -o tsv
$vnet = az resource list -g $rg --resource-type Microsoft.Network/virtualNetworks --query "[0].name" -o tsv
$appi = az resource list -g $rg --resource-type Microsoft.Insights/components --query "[0].name" -o tsv
$log = az resource list -g $rg --resource-type Microsoft.OperationalInsights/workspaces --query "[0].name" -o tsv

"Web app: $web"
"Logic App: $logic"
"Storage: $storage"
"VNet: $vnet"
"App Insights: $appi"
"Log Analytics: $log"

# 3) Security posture checks
az storage account show -n $storage -g $rg --query "{publicNetworkAccess:publicNetworkAccess,allowBlobPublicAccess:allowBlobPublicAccess,allowSharedKeyAccess:allowSharedKeyAccess,minimumTlsVersion:minimumTlsVersion}" -o table

# 4) Managed Identity on Web App
az webapp identity show -n $web -g $rg --query "{principalId:principalId,tenantId:tenantId,type:type}" -o table

# 5) Validate MI RBAC to Storage Blob Data Contributor
$webPrincipal = az webapp identity show -n $web -g $rg --query principalId -o tsv
$storageId = az storage account show -n $storage -g $rg --query id -o tsv
az role assignment list --assignee-object-id $webPrincipal --scope $storageId --query "[].{role:roleDefinitionName,scope:scope}" -o table

# 6) App settings checks (do not print full LOGIC_APP_URL in recordings)
az webapp config appsettings list -n $web -g $rg --query "[?name=='STORAGE_ACCOUNT_NAME' || name=='LOGIC_APP_URL' || name=='APPLICATIONINSIGHTS_CONNECTION_STRING'].{name:name,value:value}" -o table

# 7) Logic App trigger and Office 365 connection
az logic workflow show -n $logic -g $rg --query "{state:state,provisioningState:provisioningState}" -o table
az resource list -g $rg --resource-type Microsoft.Web/connections --query "[].{name:name,displayName:properties.displayName,statuses:properties.statuses}" -o json

# 8) Web app liveness
$url = "https://$web.azurewebsites.net"
Invoke-WebRequest -Uri $url -Method Get -UseBasicParsing | Select-Object StatusCode
~~~

## 🔐 Logic App Outlook Connector Authentication Guide

### Initial authentication (interactive sign-in required)

1. Azure Portal -> Resource Group -> open Office 365 API connection resource
   (name pattern: office365-logic-secureupload-dev-xxxxxx).
2. Open Edit API connection.
3. Click Authorize.
4. Sign in with the mailbox identity that will send OTP emails.
5. Grant consent for Office 365 Mail.Send equivalent permissions.
6. Return to connection blade and confirm status shows Connected.

### Common pitfalls and fixes

| Pitfall | Symptom | Fix |
| --- | --- | --- |
| Mailbox account lacks license | Connector fails or 401 | Use licensed M365 mailbox with Exchange Online enabled |
| Conditional Access blocks interactive login | Auth popup loops or fails | Use approved network/device; satisfy MFA/CA policy |
| Connector bound to wrong tenant/user | OTP mail never arrives | Recreate or re-authorize connection with correct mailbox identity |
| Token expired/revoked | Logic App run fails at Send email action | Re-authorize connection from API connection blade |
| Spam filtering | OTP delayed or not visible | Check Junk/Quarantine and allow sender/domain |

### Re-authentication runbook

~~~powershell
# Resolve API connection and inspect statuses
$conn = az resource list -g $rg --resource-type Microsoft.Web/connections --query "[0].name" -o tsv
az resource show -g $rg -n $conn --resource-type Microsoft.Web/connections --query "properties.statuses" -o json
~~~

Portal re-auth steps:

1. Open the API connection resource.
2. Select Edit API connection.
3. Click Reconnect or Authorize.
4. Complete interactive sign-in with OTP sender mailbox.
5. Re-run a test OTP request from web app.
6. Confirm Logic App run succeeds and email arrives.

Operational recommendation:

- Perform connector re-authentication during pre-demo, never during live audience Q and A.

## 🚨 Contingency Playbook

### Scenario 1 - OTP email not delivered

Quick diagnosis:

~~~powershell
az logic workflow show -n $logic -g $rg --query "{state:state,provisioningState:provisioningState}" -o table
az resource list -g $rg --resource-type Microsoft.Web/connections --query "[].{name:name,statuses:properties.statuses}" -o json
~~~

Recovery:

1. Re-authorize Office 365 API connection.
2. Submit new OTP request from Index page.
3. Check mailbox junk/quarantine.

Skip-and-continue guidance:

- If email remains blocked, show prior successful run history screenshot and proceed to storage/network security sections.

### Scenario 2 - Upload fails after successful OTP

Quick diagnosis:

~~~powershell
az webapp identity show -n $web -g $rg --query principalId -o tsv
az role assignment list --assignee-object-id $webPrincipal --scope $storageId -o table
az storage account show -n $storage -g $rg --query "{publicNetworkAccess:publicNetworkAccess}" -o table
~~~

Recovery:

1. Confirm Storage Blob Data Contributor role exists for web app principal.
2. If role missing, assign and wait 1-3 minutes for propagation.
3. Retry upload.

Skip-and-continue guidance:

- Demonstrate OTP and Logic App flow, then present storage IAM configuration as proof of intended access path.

### Scenario 3 - Web app cannot reach Logic App trigger

Quick diagnosis:

~~~powershell
az webapp config appsettings list -n $web -g $rg --query "[?name=='LOGIC_APP_URL']" -o table
az logic workflow show -n $logic -g $rg --query state -o tsv
~~~

Recovery:

1. Refresh LOGIC_APP_URL from workflow trigger callback URL.
2. Update app setting and restart web app.
3. Re-test OTP request.

Skip-and-continue guidance:

- If still failing, walk through portal blades and explain secure design decisions with preflight evidence.
