# Architecture Assessment — Secure OTP-Gated File Upload Portal

> Generated by architect agent | 2026-04-06

## Executive Summary

The secure file upload portal uses a layered architecture: an Azure App Service hosts the .NET 10 Razor Pages frontend, an Azure Logic App (Consumption) handles email delivery via the Office 365 connector, and Azure Blob Storage (private endpoint, managed identity) secures file uploads. All components are connected through a VNet with private DNS resolution, eliminating public storage exposure.

## Security Architecture

### Identity & Access

| Access Path                  | Mechanism                              | Notes                                        |
| ----------------------------- | -------------------------------------- | -------------------------------------------- |
| App Service → Storage Blob   | System-Assigned Managed Identity       | `Storage Blob Data Contributor` role         |
| App Service → Logic App      | HTTPS POST to SAS URL                  | URL stored in app setting `LOGIC_APP_URL`    |
| Developer → Storage (deploy) | Deployer principal role assignment     | `Storage Blob Data Contributor` on deployer  |
| Logic App → O365 Outlook     | O365 API Connection (OAuth)            | Configured at deploy; user consents          |

> [!IMPORTANT]
> The Logic App SAS URL contains a shared-access signature issued by Azure. While the URL is not a secret in the traditional sense, it should be treated as sensitive and stored only in App Service App Settings (not in source control). The demo documents this pattern explicitly.

### Network Security

| Layer               | Control                                                              |
| ------------------- | -------------------------------------------------------------------- |
| Storage Account     | Public network access: Disabled; accessible only via private endpoint|
| App Service         | VNet Integration to `snet-appservice-dev`; routes blob traffic privately|
| Private DNS Zone    | `privatelink.blob.core.windows.net` resolves to private IP in VNet  |
| Logic App           | Public endpoint (Consumption plan); secured by SAS in trigger URL   |
| Communication       | TLS 1.2 minimum on all services; HTTPS-only on App Service          |

### OTP Security Model

| Control                     | Implementation                                          |
| ---------------------------- | ------------------------------------------------------- |
| OTP generation               | `RandomNumberGenerator` (CSPRNG) — not `Random`        |
| OTP storage                  | Server-side session (in-memory, never persisted)        |
| OTP lifetime                 | 10-minute expiry tracked by session timestamp           |
| Rate limiting                | Max 5 failed attempts → session reset + re-enter email  |
| Transport                    | Email only — OTP never appears in HTTP response body   |

## Runtime Flow

### OTP Request Flow

```
1. User → App Service /         (enters email address)
2. App Service → Logic App      (POST to SAS URL: {email, otp})
3. Logic App → O365 Connector   (Send Email action)
4. O365 Connector → User email  (delivers OTP)
5. User → App Service /verify   (submits OTP)
6. App Service                  (validates OTP vs. session; checks expiry)
7. App Service                  (marks session authenticated; redirects to /upload)
```

### File Upload Flow

```
1. User → App Service /upload   (authenticated; submits file)
2. App Service → Blob Storage   (DefaultAzureCredential → Managed Identity)
3. Blob Storage                 (stores file in uploads container; returns blob URI)
4. App Service → User /success  (confirmation with filename)
```

## Architecture Decision Records Summary

| Decision                        | Choice                             | Rationale                                                           |
| ------------------------------- | ---------------------------------- | ------------------------------------------------------------------- |
| OTP sending mechanism           | Azure Logic App + O365 connector   | No SMTP credentials needed; enterprise Office 365 connector         |
| App → Logic App auth            | SAS URL in app setting             | Simplest secure pattern for Consumption Logic Apps; no AAD required |
| Storage access                  | Managed Identity                   | Eliminates storage key rotation; industry best practice             |
| Storage network                 | Private Endpoint + DNS Zone        | Demonstrates enterprise-grade isolation; no public blob exposure    |
| OTP persistence                 | In-memory session                  | No database dependency; sufficient for demo lifespan                |
| App Service SKU                 | B2 (Linux)                         | Low cost for demo; supports VNet integration                        |
| Logic App plan                  | Consumption                        | Pay-per-trigger; zero idle cost                                     |
| IaC strategy                    | Bicep + AVM                        | AVM-first policy; accelerated and validated module library          |
